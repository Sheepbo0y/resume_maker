# ATS-friendly Resume Template Guidelines

- 目的：确保简历模板能被 ATS（Applicant Tracking Systems）解析，并在招聘系统中正确抓取信息。
- 范围：ClassicProfessional、ModernMinimal、CreativeSimple 三种模板的结构、语义化标签、可访问性与打印友好性。

## 1. 结构与语义
- 使用语义化 HTML 标签：header、main/section/article、nav、footer、address、ul/li、time 等。
- 顶级标题层次：在文档中应有一个主标题（h1，通常为姓名），随后是清晰的二级标题（h2）如 Summary、Experience、Education、Projects、Skills 等。
- 避免使用复杂表格和图片来表示关键信息，信息应以文本和列表形式呈现。
- 联系信息推荐使用 address/a 标签（mailto/tel），确保可索引性。
- 打印友好：确保在打印时信息仍然清晰可读，尽量避免花哨的布局和背景颜色。

- 示例要点：
  - 姓名放在 h1，紧接着放联系信息。
  - Summary 放在一个独立的 section，带有明确的 aria-labelledby。
  - Experience、Education、Skills、Projects 使用 h2，且顺序自上而下一致。

## 2. 可访问性 (Accessibility)
- 为每个信息区提供明确的标签，如 aria-labelledby 指定的标题。
- 联系信息使用 address 标签，包含可点击的 mailto/tel 链接。
- 颜色对比遵循 WCAG 2.1 AA，避免低对比度的组合。
- 键盘导航可用，信息结构保持直观的阅读顺序。
- 屏幕阅读器友好：没有依赖仅由图片承载的文本信息，尽量用文本标注数据。

## 3. ATS 兼容性要点
- 不要使用嵌入式对象、脚本驱动的内容或隐藏文本来呈现数据。
- 使用简单的列表呈现技能、项目等要点，方便 ATS 解析。
- 保证同一信息在不同模板中可被一致解析（姓名、邮箱、电话、地点、工作经历、教育经历、技能等字段要可识别）。
- 使用简洁的日期格式，例如 2019-2021 或 2019 – 2021，避免复杂日期描述。

## 4. 常见错误与避免
- 使用复杂的嵌套结构，造成 ATS 解析困难。
- 依赖图片代替文本信息（如名称图片、图标等）。
- 低对比度颜色导致文本不可读。
- 搭配大量自定义样式导致文本提取困难。
- 在关键字段使用非一致的标签名称（例如所有部分都使用自定义类名而非语义标签）。

## 5. 最佳实践与验证
- 使用 HTML 验证工具检查语义结构。
- 通过 ATS 的简历导入预览测试字段提取是否正确（名称、邮箱、电话、职位、公司、地点等字段是否可识别）。
- 打印预览测试，确保文本可打印且可读。

## 验证要点
- 确认模板中的每个信息区都使用语义标签且有清晰标题。
- 联系信息可用 mailto/tel 链接。
- 打印输出无隐藏信息，且文本信息完整。
